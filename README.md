## 🌿 About the Project

**AyurSutra** is a full-stack digital platform that modernizes and digitizes Panchakarma patient management for Ayurvedic clinics and hospitals. Developed for **Smart India Hackathon 2025** (Problem Statement ID: **SIH25023**), it bridges the gap between ancient Ayurvedic wisdom and modern healthcare technology.

Traditional Panchakarma management relies on manual scheduling, paper records, and verbal queue updates — causing inefficiency, long wait times, and lack of transparency. AyurSutra solves this with AI-powered scheduling, real-time smart queues, blockchain-backed records, and an inclusive multilingual voice interface.

### The Problem It Solves

| Pain Point | AyurSutra Solution |
|---|---|
| Long, unpredictable wait times | Live queue with real-time ETA updates |
| Manual, error-prone scheduling | AI-based Dosha + availability slot allocation |
| Opaque therapy progress | Visual therapy roadmap (Digital Twin) |
| Paper records prone to tampering | Blockchain-backed health ledger |
| Inaccessible for rural/elderly patients | Multilingual voice booking interface |
| No emergency prioritization | Fast-Track Protocol with auto-notifications |

## 📊 Feasibility & Viability

### ✅ Feasibility Analysis

| Dimension | Assessment |
|---|---|
| **Technical** | Proven stack (React, Node.js, PostgreSQL, Blockchain) — scalable and widely supported |
| **Operational** | Fits directly into existing clinic workflows: appointments, therapy management, monitoring |
| **Economic** | Cloud-based pay-per-use model minimizes upfront infrastructure costs |
| **Social** | Multilingual support and voice interface ensure inclusivity across urban & rural India |

### ⚠️ Challenges & Mitigation

| Challenge | Mitigation Strategy |
|---|---|
| Data privacy & HIPAA compliance | End-to-end encryption, blockchain logs, role-based access |
| Adoption resistance from staff | Easy-to-use UI, onboarding workshops, training resources |
| Legacy system integration | Standardized APIs (FHIR/HL7) for interoperability |
| Scalability under high load | Auto-scaling cloud clusters for peak traffic |
| AI bias or scheduling errors | Human-AI collaboration — therapist reviews AI suggestions before confirmation |

## 🚀 Getting Started

### Prerequisites

- Node.js v18+
- PostgreSQL 15+
- Docker & Docker Compose
- Firebase project (for real-time features)

### Installation

```bash
# Clone the repository
git clone https://github.com/Fakemithu/AyurSutra.git
cd AyurSutra
```

```bash
# Install dependencies
npm install

# Set up environment variables
cp .env.example .env
# Edit .env with your DB, Firebase, and blockchain config
```

```bash
# Run database migrations
npm run migrate

# Start development server
npm run dev
```

### Using Docker

```bash
# Build and run all services
docker-compose up --build
```

The app will be available at `http://localhost:3000`

### Literature Survey Findings

1. Existing Ayurveda digital systems are **workflow-first, not intelligence-first** — they lack AI-driven optimization.
2. AI in healthcare scheduling shows proven value but **requires clinician oversight** and transparency.
3. Blockchain for medical records has been explored but **practical deployments remain limited** due to performance trade-offs.

### Identified Research Gaps (Addressed by AyurSutra)

- ❌ No Ayurveda-specific system unifying therapy progress, real-time queues, and Dosha-based AI slotting
- ❌ Auditability of therapy records not strongly guaranteed across vendors
- ❌ Minimal linkage between lifestyle/wearable signals and therapy scheduling
- ❌ Limited inclusive access (multilingual voice) for rural and elderly patients
